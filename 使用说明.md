# TikTok Shop 商品上架工具 — 使用说明

> 店小秘批量上架表格生成器

本工具用于将 ERP 导出的商品表格一键转换为**店小秘 TikTok Shop 批量上架模板**（.xlsx），自动完成定价计算、变种属性拆分和类目匹配，大幅减少手动填表的工作量。

## 1. 使用前准备

### 1.1 文件要求

- 文件来源：**ERP 系统导出的 .xls 文件**
- 文件格式：实际上是 HTML 表格保存为 `.xls` 扩展名（这是大多数 ERP 系统的默认导出格式）
- 文件内容：第一行为列头（40 列），后续行为商品数据，每一行代表一个 SKU 变体
- 不支持原生 Excel 二进制格式（.xlsx / .xlsb），如遇格式错误请确认是 ERP 直接导出的文件

### 1.2 需要提前确认的信息

| 信息 | 说明 | 示例 |
|------|------|------|
| **仓库名称** | 必填，必须与店小秘后台配置的仓库名称**完全一致** | `深圳仓` |
| **品牌名称** | 可选，默认为「无品牌」 | `无品牌` |
| **目标国家** | 决定定价公式和币种 | 菲律宾(PH)、英国(GB)等 |
| **利润率** | 定价利润百分比 | 28%（东南亚默认） |
| **折扣率** | TikTok 展示折扣百分比 | 45%（东南亚默认） |

### 1.3 网络要求

工具运行需要联网访问以下服务：

- **汇率接口** — 自动获取最新美元兑各币种汇率
- **AI 分析** — 用于变种属性拆分和类目兜底分析
- **TikTok API** — 用于查询商品推荐类目

> 商品数据本身在浏览器端处理，不会上传到外部服务器。只有产品名称和属性信息会在 AI 分析和类目查询时发送。

## 2. 操作步骤详解

工具采用 **6 步向导式流程**，按顺序操作即可。页面顶部的步骤导航条显示当前进度，已完成的步骤可以点击返回。

---

### 步骤 1：上传数据

**目的：** 上传 ERP 导出文件并配置基本信息。

**操作方法：**

1. **上传文件** — 将 `.xls` 文件拖拽到上传区域，或点击上传区域选择文件
2. **查看解析结果** — 上传成功后，页面会显示：
   - 数据总行数
   - 不同产品数量（按 ERP ID 去重统计）
   - 列数
3. **填写配置信息：**
   - **默认品牌名称** — 输入产品品牌，留空则使用「无品牌」
   - **仓库名称（必填）** — 输入店小秘中配置的仓库名称，此字段不能为空
4. **检查数据预览** — 下方表格展示所有数据行，不同产品用交替色块区分（按 ERP ID 分组），支持翻页浏览

**提示：**
- 如果提示「暂不支持此文件格式」，请确认上传的是 ERP 直接导出的 `.xls` 文件
- 如果提示「文件中未找到数据行」，请确认文件中包含表头和数据

---

### 步骤 2：列映射

**目的：** 配置 ERP 源列与店小秘模板目标列的对应关系。

**默认已配置好**常用映射，通常无需修改。映射表包含 35 行（对应模板的 35 列），每行显示：

| 列 | 说明 |
|----|------|
| 目标列 | 店小秘模板的列名，带 `*` 的为必填列 |
| 源数据列 | 对应的 ERP 列，下拉选择 |
| 转换方式 | 数据处理方式 |
| 固定值 / 说明 | 固定值或备注信息 |

**转换方式说明：**

| 转换方式 | 含义 | 使用场景 |
|---------|------|---------|
| 直接映射 | 原样复制 | 标题、SKU、库存等 |
| ÷1000 (g→kg) | 数值除以 1000 | 重量从克转公斤 |
| HTML→文本 | 去除 HTML 标签 | 需要将富文本转纯文本的字段 |
| 固定值 | 所有行填相同值 | 品牌=无品牌、货到付款=是 |
| 定价公式 | 步骤 4 自动计算 | 本地展示价 |

**关键默认映射一览：**

| 目标列 | 源数据列 | 转换 |
|--------|---------|------|
| 产品标题 | 商品名称1 | 直接映射 |
| 产品描述 | 商品描述 | 直接映射 |
| 品牌 | — | 固定值：无品牌 |
| SKU | Seller SKU | 直接映射 |
| 变种属性名称一 | — | 固定值：Color |
| 变种属性值一 | 规格1（颜色） | 直接映射 |
| 变种属性名称二 | — | 固定值：Size |
| 变种属性值二 | 规格2（尺寸） | 直接映射 |
| 本地展示价 | — | 定价公式 |
| 库存 | 全球商品库存 | 直接映射 |
| 产品主图 | 商品主图 | 直接映射 |
| 附图一~八 | 商品图片2~9 | 直接映射 |
| 重量(kg) | 商品重量(g) | ÷1000 |
| 长/宽/高(cm) | 商品长度/宽度/高度(cm) | 直接映射 |
| 仓库名称 | — | 固定值（步骤 1 中填写） |
| 货到付款 | — | 固定值：是 |

**提示：** 默认情况下，产品描述会保留 ERP 源数据中的 HTML 代码（如 `<br>`、`<img>`）。只有当你的 ERP 导出列名与默认配置不同时才需要修改映射。

### 步骤 3：变种分析

**目的：** 用 AI 自动识别并拆分产品的变种维度（如颜色、尺寸）。

**适用场景：** 当产品的「产品属性」字段包含组合值（如 `黑色-XS`、`蓝色-L`），需要拆分为独立的变种维度供 TikTok 使用。

**处理逻辑：**

工具会将产品分为三类，分别处理：

| 类型 | 条件 | 处理方式 |
|------|------|---------|
| 已有规格列 | 规格1（颜色）和规格2（尺寸）已有数据 | 跳过分析，直接使用原有数据 |
| 待分析 | 「产品属性」字段有组合值 | 发送给 AI 拆分 |
| 无变种 | 单一 SKU，无属性值 | 无需操作 |

**操作方法：**

1. 进入步骤 3 后，AI 会**自动开始批量分析**需要处理的产品
2. 分析进度和当前处理的产品名称会实时显示
3. 分析完成后，查看结果表格中每个产品的维度拆分结果

**手动干预：**

- **停止分析** — 点击「停止」按钮中断批量处理
- **单个重新分析** — 对特定产品点击「重新分析」
- **重置** — 清除分析结果，恢复为原始规格列数据

**示例：**

```
产品属性: ["黑色-XS", "黑色-S", "蓝色-M"]
   ↓ AI 分析结果
维度 1: Color (Black, Blue)
维度 2: Size (XS, S, M)
```

**提示：** 如果看到「存在空白属性值」的橙色警告，建议重新分析该产品或手动修正。

### 步骤 4：定价设置

**目的：** 配置目标国家和售价计算参数。

#### 4.1 选择站点国家

支持 11 个 TikTok Shop 站点，分三个区域：

| 区域 | 国家 | 默认利润率 | 默认折扣 |
|------|------|-----------|---------|
| 东南亚 | 菲律宾(PH)、马来西亚(MY)、新加坡(SG)、泰国(TH)、越南(VN) | 28% | 45% |
| 欧洲 | 英国(GB)、德国(DE)、法国(FR)、意大利(IT)、西班牙(ES) | 35% | 42% |
| 墨西哥 | 墨西哥(MX) | 30% | 40% |

> 切换国家会自动恢复该国家的默认利润率和折扣值。

#### 4.2 可调节参数

- **定价利润率** — 希望获得的利润百分比，可根据实际需求调整
- **折扣率** — TikTok 上展示的折扣幅度，最终导出的是折前原价（即「划线价」）

#### 4.3 汇率信息

- 汇率在页面加载时**自动获取**，缓存 60 分钟
- 如需手动刷新，点击「刷新汇率」按钮
- 汇率状态显示：USD/CNY、USD/站点币种 的实时数值
- 如汇率获取失败，会显示红色警告，此时定价功能暂不可用

#### 4.4 价格预览

页面下方显示前几条数据的**完整定价分解**，包括：

- 成本（CNY）
- 重量（克 / 公斤）
- 处理费（0.8 元）
- 跨境物流费（站点币种）
- 折后售价（站点币种）
- 折扣率
- **导出折前售价**（最终导入店小秘的价格）

> 请核对预览价格是否合理，再进入下一步。

#### 4.5 欧洲站点特别说明

- 产品形态分为**普货、特货、敏货**三类，影响运费和 VAT 税率
- 产品形态读取源数据中的「产品形态」列，如未标注则默认按「普货」计算
- 英国站(GB)特殊规则：折后售价达到 £35 时会额外加收 £3.99

### 步骤 5：推荐类目

**目的：** 为每个产品匹配 TikTok Shop 分类 ID。

**操作方法：**

1. 点击**「批量获取推荐类目」**按钮
2. 系统会逐个处理每个产品（按 ERP ID 去重），处理逻辑：
   - **第一步** — 调用 TikTok 官方 API，根据产品标题推荐类目
   - **第二步** — 如果 API 未匹配到结果，自动切换到 AI 分析
3. 处理结果在表格中显示：
   - **分类 ID** — TikTok 上的类目编号
   - **类目来源** — 标注「TikTok 推荐」或「AI 分析」
   - **状态** — 已填写 / 待确认 / 待处理 / 待填写

**手动操作：**

- **手动输入类目 ID** — 如果知道正确的类目 ID，直接在输入框中填写
- **单个 AI 分析** — 对特定产品点击「AI 分析」重新匹配
- **AI 候选** — AI 返回多个候选类目时，点击「应用」选择合适的一个

**提示：** 类目 ID 不是必须在本步骤填写，可以导出后在店小秘中手动补充。

## 3. 定价逻辑简介

工具根据选择的目标国家，使用不同的定价模型。核心逻辑是在产品成本上叠加运费、平台费用、税费和利润，最终计算出店铺展示的「划线价」。

### 通用流程

```
商品成本(CNY) + 0.8元处理费
        ↓
  转换为站点当地货币（通过美元汇率）
        ↓
  加上跨境物流费（按重量计算）
        ↓
  除以 (1 - 利润率 - 平台费率)，加上税费
        ↓
  得到「折后售价」（买家实际支付价）
        ↓
  折后售价 ÷ (1 - 折扣率)
        ↓
  得到「折前售价」← 这就是导出到模板的价格
```

### 三种定价模式

#### 东南亚模式（PH / MY / SG / TH / VN）

- 运费按阶梯计算：起步重量 + 每 0.01kg 续费
- 包含平台综合费率（20%~26%，含 5% 达人佣金）
- 部分国家有额外税费系数（如马来西亚 ×1.1、泰国 ×1.224）

**公式：**
```
成本(站点币种) = ((成本 + 0.8元处理费) / USD÷CNY) × USD÷站点币种
折后售价 = ((成本 + 跨境物流) / (1 - 利润率 - 综合费率)) × 税费系数
折前售价 = 折后售价 / (1 - 折扣)
```

#### 欧洲模式（GB / DE / FR / IT / ES）

- 运费按连续公斤计算：起步运费（0.05kg 内）+ 超重部分 × 每公斤续费
- 运费和 VAT 根据产品形态（普货/特货/敏货）不同
- VAT 从含税价反向推算
- 英国(GB)特殊：折后售价 ≥ £35 时加收 £3.99

**公式：**
```
成本(站点币种) = ((成本 + 0.8元处理费) / USD÷CNY) × USD÷站点币种
跨境物流 = 起始运费 + max(重量kg - 0.05, 0) × 每公斤续费
税前价 = (成本 + 跨境物流) / (1 - 利润率 - 平台佣金率)
VAT = (税前价 + 3.99) × VAT率 / (1 + VAT率)
折后售价 = 税前价 + VAT [+ UK附加费]
折前售价 = 折后售价 / (1 - 折扣)
```

#### 墨西哥模式（MX）

- 运费按重量分档（0-1kg、1-2kg、2-5kg、5-30kg），不同档位不同基础费
- SFP 物流补贴 59 MXN 从运费中扣减
- 包含进口税计算（税率 33.5%，免税额 4.5 USD，最低税额 6 MXN）
- 综合费率 29%（含平台费、SFP、电商税、达人佣金、广告）

**公式：**
```
成本(站点币种) = ((成本 + 0.8元处理费) / USD÷CNY) × USD÷MXN
跨境物流 = max(重量kg × 每公斤费 + 基础费 - 59补贴, 0)
税前价 = (成本 + 跨境物流) / (1 - 平台费率 - 利润率)
进口税 = max((税前价×1.335 + 59 - 免税额×汇率) / 1.335 × 0.335, 6)
折后售价 = 税前价 + 进口税
折前售价 = 折后售价 / (1 - 折扣)
```

### 重量来源

定价使用的重量优先读取「发货重量（有差值）」列，如该列为空则退回「商品重量(g)」列。重量单位为克，公式内部会自动转换为公斤。

## 4. 常见问题

### Q：上传文件提示「暂不支持此文件格式」
**A：** 请确认文件是 ERP 系统直接导出的 `.xls` 文件。如果是在 Excel 中另存为的 `.xlsx` 文件，不被支持。需要使用 ERP 原始导出的 HTML 格式 `.xls`。

### Q：价格预览显示「—」
**A：** 可能的原因：
- 该产品缺少成本或重量数据 → 检查 ERP 源文件对应字段
- 汇率获取失败 → 点击「刷新汇率」重试
- 利润率或折扣设为 0 → 检查步骤 4 的参数设置

### Q：汇率获取失败怎么办？
**A：** 检查网络连接后点击「刷新汇率」。系统会使用最近一次成功获取的缓存汇率（页面会提示缓存时间）。如果一直失败，定价功能暂不可用，但其他步骤可以正常操作。

### Q：AI 变种分析结果不准确
**A：**
- 对单个产品点击「重新分析」让 AI 再试一次
- 如果多次不准确，可以点击「重置」恢复到原始规格列数据
- 也可以导出后在 Excel 中手动修改变种属性

### Q：类目查询很多产品都失败了
**A：** TikTok API 失败后会自动切换到 AI 分析。仍然失败的产品可以：
- 单独点击「AI 分析」重试
- 手动在输入框中填写已知的分类 ID
- 导出后在店小秘中手动选择类目

### Q：导出的文件无法导入店小秘
**A：** 常见原因：
- **仓库名称不匹配** — 确保与店小秘后台配置的名称完全一致（包括空格和标点）
- **必填字段为空** — 检查步骤 6 中标橙色的缺失项
- **分类 ID 缺失** — 分类 ID 必须填写，可在步骤 5 补充或导出后手动填写

### Q：产品有 3 个以上变种维度怎么办？
**A：** 模板最多支持 3 个变种维度。本工具处理前 2 个维度（通常是颜色和尺寸），第 3 维度需要在导出文件中手动填写「变种属性名称三」和「变种属性值三」列。

### Q：导出的重量或尺寸值不对
**A：** 检查步骤 2 列映射中的配置：
- 重量默认从「商品重量(g)」列读取并自动 ÷1000 转为公斤
- 长/宽/高从「商品长度/宽度/高度(cm)」列读取，单位为厘米
- 如果 ERP 中列名不同，需在步骤 2 中修改对应的源数据列

### Q：可以同时导出多个国家的定价吗？
**A：** 目前每次只能选择一个国家导出。如需多国定价，可以在步骤 4 切换国家后重新回到步骤 6 导出，每次导出一个国家的文件。

---

## 5. 注意事项

1. **仓库名称必须精确匹配** — 这是导入店小秘时最常见的失败原因，名称必须与店小秘后台配置完全一致
2. **汇率实时波动** — 汇率来自公开接口，建议在批量上架前确认价格是否合理
3. **AI 结果需要抽检** — 变种分析和类目推荐由 AI 生成，建议对关键产品手动复查
4. **大批量数据处理较慢** — 步骤 3（变种分析）和步骤 5（类目查询）需要逐个调用接口，产品数量多时请耐心等待
5. **浏览器端处理** — 商品数据在本地浏览器中处理，刷新页面会丢失所有进度，请在操作过程中避免刷新
